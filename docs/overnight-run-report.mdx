## 🔍 The arc — what was broken, and the three fixes that unlocked it

The mesh could always *coordinate* agents. It couldn't *finish* work. Three root fixes changed that.

### 1. Worker child-process leak (#122 → fixed in #124)

Each `claude` worker spawned a subprocess tree; on timeout only the direct child was killed, so
grandchildren orphaned and piled up until they exhausted the account's session capacity — and then
*new* workers hung too. A self-strangling collapse. **Fix:** run the child in its own process group
and kill the whole group on cancel. Validated: **zero orphaned workers all night.**

### 2. Planner over-decomposition (fixed in #124)

Opus split modest tickets into 10–13 tasks, multiplying load on the single reviewer and widening every
cascade failure. **Fix:** default to **one task**, split only for genuinely independent work.
Result: **13/13 jobs = 1 task.**

### 3. Worker "Definition of Done" hardening (#130) — **the unlock**

Workers did correct work, but the Opus reviewer kept issuing `request_changes` for fixable nits
(an unaligned `gofmt` line, a feature left backend-only), and `main` has no auto-retry, so good work
hard-failed. **Fix:** the worker now runs `gofmt` / `go build` / `go test` and re-checks every
acceptance criterion before finishing. Jobs flipped from nit-failing to **first-try approvals — four
clean passes in a row** right after it landed.

## ⚠️ Main issues encountered (operational)

1. **Cold-start review race (#125)** — the biggest recurring cost. A freshly auto-spawned reviewer
   misses its first re-delivered request; that job then burns the full 20-minute gate before failing.
   Hit twice (#97, #111-retry). Mitigation used: keep the reviewer warm (no bounces).
2. **`request_changes` = hard fail (#85 gap)** — `main` has no auto re-dispatch with feedback, so
   good-but-imperfect work dies on the first nit. Worked around by hardening the worker (fix #3).
3. **Single-reviewer throughput ceiling (#123)** — one Opus reviewer serializes every review, capping
   the fleet at 2 workers all night. This is the next unlock.
4. **Shared-file merge conflicts** — parallel jobs editing `config.go` / `web/app.js` collided at merge
   time (claims are advisory within a run, not across git branches). Resolved by sequential rebasing
   and keep-both merges.
5. **Director self-inflicted hiccup** — a `pkill -f` pattern matched the director's own shell and killed
   it mid-bounce. Recovered immediately; lesson logged: kill the coordinator by explicit PID only.
