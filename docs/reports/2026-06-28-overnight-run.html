Two nights earlier the same fleet merged nothing — it could coordinate agents but collapsed before finishing real work. That run produced a bug report, not code. This run closed the loop.

What was broken — and the three fixes that unlocked it

1

The worker leaked its own children fixed · #122

Each claude worker spawned a subprocess tree; on timeout only the direct child was killed, so grandchildren orphaned and piled up until they exhausted the account's session capacity and new workers hung too — a self-strangling collapse. Fix: run the child in its own process group and kill the whole group on cancel.

2

The planner over-decomposed

Opus split modest tickets into 10–13 tasks, multiplying load on the single reviewer and widening every cascade failure. Fix: the planner now defaults to one task and only splits for genuinely independent work.

3

Workers shipped good code that failed review on nits fixed · #130

This was the unlock. Workers did correct work but the Opus reviewer kept issuing request_changes for fixable details — an unaligned gofmt line, a feature left backend-only — and main had no auto-retry, so good work hard-failed. Fix: a Definition of Done — the worker now runs gofmt / go build / go test and re-checks every acceptance criterion before finishing. Jobs flipped from nit-failing to first-try approvals: four clean passes in a row right after it landed.

Bugs the run surfaced

#122
Worker child-process leakfixed — grandchild processes orphaned on timeout. Fixed via process-group kill.
#123
Single-reviewer throughput ceiling — one Opus reviewer serializes every review; needs an expert pool to scale past ~2 workers.
#125
Cold-start review race — a freshly auto-spawned reviewer can miss the re-delivered first request. Warm reviewers are unaffected.
#85
No auto re-dispatch on request_changes — on main, requested changes hard-fail instead of looping back with feedback. Worked around by hardening the worker so it ships review-clean.

Honest notes

·

Cold-start cost two reviews

Each time the reviewer spawned cold, its very first review could be lost to #125. Keeping the reviewer warm (no coordinator bounces) made every subsequent review land.

·

Dashboard jobs collided on web/app.js

Parallel UI jobs edit the same file and conflict at merge time — claims are advisory within a run, not across git branches. Resolved by sequencing UI work and keep-both rebases.

·

Throughput stayed at 2 workers

The single reviewer (#123) is the ceiling. The fix list is clear: an expert pool is the next unlock.
