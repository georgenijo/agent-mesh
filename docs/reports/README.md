# Reports

Dated, point-in-time reports — build milestones and autonomous-run writeups.
**Convention:** one report per event, named `YYYY-MM-DD-<slug>.<ext>`. A report
is a snapshot, frozen at its date; later runs get their own file rather than
editing an old one. An `.mdx`/`.html` pair is the same report in two forms
(source doc + standalone render).

| Date | Report | Form |
| --- | --- | --- |
| 2026-06-05 | `2026-06-05-p0-build-report.md` — P0 presence walking skeleton | md |
| 2026-06-05 | `2026-06-05-p1-build-report.md` — P1 conflict avoidance + blackboard | md |
| 2026-06-28 | `2026-06-28-overnight-run.{mdx,html}` — first unattended autonomous run | mdx + html |
| 2026-06-30 | `2026-06-30-v1-v2-before-after.{mdx,html}` — V1 → V2 iteration writeup | mdx + html |
