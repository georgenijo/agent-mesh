Agent Mesh · Iteration Report

V1 → V2: from a spine you drive to a fleet that drives itself

Window 2026-06-23 → 2026-06-30  ·  Repo georgenijo/agent-mesh  ·  Mode autonomous

V1 helped agents you ran avoid each other. V2 runs the agents for you. Drop a ticket → a Sonnet worker builds it → an Opus expert reviews it → it merges. No human in the build loop.

Before · V1

A CLI fabric — mesh join/claim/ask/note — so coding agents you drive by hand discover each other, avoid editing the same files, and ask each other questions. You are the operator.

After · V2

A hands-off service. Drop a GitHub issue (or say "work on issues 110–115"); the coordinator triages, auto-spawns experts + workers, builds in an isolated worktree, an Opus reviewer gates the merge, and the dashboard streams it live. You watch.

Proof · overnight run

The 06-28 run cleared the backlog unattended: 10 issues + 2 infra fixes, 14 PRs, ~$26, main never broke. Prior run (06-26): 0 / 30. The fleet even fixed 5 of its own bugs.

The pivot in one table

Eleven dimensions, before and after. The only one that didn't move: cognition still runs on the user's CLI subscription — no API key in core.

Who drives
V1The human. Agents work solo ~95% of the time; mesh assists only on overlap. Collaboration is opt-in per message.
V2The system. You drop a ticket; the coordinator triages, spawns a team, executes, reviews, reports. You watch.
Intake
V1A human runs mesh verbs at the terminal.
V2GitHub issue link, repo-picker dropdown, natural language ("work on issue N / range / all"), or POST /jobs. One mesh up brings the fleet up armed.
Agent model
V1Long-lived agents the mesh merely assists; no team or pipeline concept.
V2Hybrid: warm persistent experts (hold the codebase map, answer + review) paired with ephemeral workers (one job, then exit).
Expert lifecycle
V1One manually-started responder loop (mesh expert serve).
V2Coordinator auto-spawns an expert for any un-owned role; a reviewer pool parallelises reviews; idle experts self-terminate on a TTL.
Worker execution
V1N/A — no worker concept.
V2Isolated git worktree-per-task, embedded per-worker sidecar for real mesh claim/ask --wait, auto-commit onto the task branch.
Quality gate
V1None in the original design (P2 was async ask/answer).
V2Every diff is routed to a reviewer; the typed verdict gates the task. Workers self-verify a Definition of Done; request_changes → bounded re-dispatch with feedback; ambiguity → escalate, not guess.
Cognition source
V1CLI invocation on the user's subscription; no API key in core.
V2Same. Every brain is claude -p / codex exec, headless + structured output. The invariant survived the pivot intact.
Transport
V1Assumed NATS/JetStream as a separate process (early framing).
V2Coordinator-embedded star bus over a unix socket (pub/sub + KV CAS/TTL + JSONL streams). bus.Client is the NATS swap seam — no distributed tax paid early.
Coordinator role
V1Registry + presence + ask routing.
V2Control plane expanded: triage (planner → task DAG), schedule, review gate, budget metering (hard cap, pause-not-fail), idle reaping, audit fan-out.
Blackboard
V1Per-repo decision streams a human replays with mesh context.
V2Same streams, now the experts' long-term memory (compacted primer re-injected on restart) and mesh-wide in the dashboard (aggregated across all repos).
Dashboard
V1Read-only SSE observer — presence, claims, notes.
V2Live drill-in: click a job/task/agent to stream its output, model + cumulative cost, a cost ledger vs budget, group-by-job, issue links, and a Claude-session telemetry panel into each worker's transcript.

What shipped this week, by theme

06-23 → 06-30. Five themes, anchored to the merged PRs.

1 · English-in intake — you stop typing paths

The front door changed from "run mesh verbs" to "tell it what you want."

  • #134 submit a job from a GitHub issue link
  • #133 natural-language control: "work on issue 110", "issues 110–115", "all open issues"
  • #138 submit-job repo picker (dropdown from MESH_REPOS_DIR)
  • #135 generic POST /jobs HTTP ingress on meshd
  • #139 arm-once drop-and-run — a single mesh up

2 · Autonomous experts — the pool scales itself

The control plane provisions and reaps its own cognition.

  • #118 on-demand experts — coordinator spawns an expert the instant an ask/review hits an un-owned role
  • #146 reviewer pool (MESH_REVIEW_POOL_SIZE) parallelises reviews, lifting the single-reviewer ceiling
  • #129 idle reaper — experts self-terminate after an inactivity TTL
  • #148 fix the cold-spawn re-deliver race (readiness signal so the first review never drops)

3 · The review loop closes — work actually finishes

V1 could coordinate agents; it couldn't finish work. Three changes flipped that.

  • #130 worker "Definition of Done" — the unlock: self-verify gofmt / build / test / acceptance before declaring done. Flipped nit-failures into four first-try approvals in a row
  • #149 bounded re-dispatch — on request_changes, re-dispatch with the reviewer's feedback instead of hard-failing
  • #150 escalation primitive — a worker hitting real ambiguity escalates rather than guessing

4 · Observability — workers stop being black boxes

See which task is running, what it's doing, and what it costs — live.

  • 8baa2c3 click drill-in + live worker streaming (state, role, model, PID, cwd, cost, branch + live log)
  • 0c77afb Claude Code session telemetry panel — correlates each cc-* worker with its native transcript
  • #136 per-agent model + cumulative cost
  • #128 / #131 persistent cost ledger + the cost-window UI
  • #127 group tasks by job with readable labels  ·  #126 GitHub issue link on jobs
  • 8bb2220 MESH_DASHBOARD_ALLOWED_HOSTS — tailnet viewing (loopback stays default)  ·  #116 mesh-wide notes

5 · Dogfood self-heal — the fleet fixed its own bugs

The mesh ran its own hardening backlog; the workers produced audit-grade, CI-verified fixes.

  • #124 the two collapse-causers: worker child-process leak (process-group kill → zero orphans) + planner over-decomposition curb (default to one task → 13/13 jobs = 1 task)
  • bb1ff57 worker tool-use unblock — --dangerously-skip-permissions so it can edit inside its bounded worktree
  • baddd72 resident-expert startup deadlock fix (lazy session-id capture)
  • 5 fleet-found fixes: KV reads no longer create buckets · sidecar no longer self-accepts its own role-ask · NaN/Inf budget rejected · MESH_DIR/MESH_REPOS_DIR absolutized · --json honored on setup errors
