## 1. VERDICT: Rebuild-Core / Immediate-Harvest

Choose **Rebuild-Core / Immediate-Harvest**: agent-mesh should remain a standalone coordination product, but it should harvest hangar's driver/session-lifecycle patterns immediately, not "later." Do **not** Layer on hangar's REST API and do **not** Merge agent-mesh into hangar Phase 9. Hangar is a strong terminal-session command center: backend owns PTYs, driver traits, an event bus, SQLite/ring history, and human-facing REST/WS control (`/Users/george-mac-mini/Documents/code/hangar/docs/ARCHITECTURE.md:60-71`, `/Users/george-mac-mini/Documents/code/hangar/docs/ARCHITECTURE.md:126-137`). Agent-mesh's hardest unsolved work is not "keep a process alive"; it is durable coordination: one envelope, one authority, CAS/TTL claims, async ticket FSM, role routing, blackboard replay, and coordinator policy (`/Users/george-mac-mini/Documents/code/agent-mesh/docs/components.md:9-28`, `/Users/george-mac-mini/Documents/code/agent-mesh/docs/components.md:45-68`, `/Users/george-mac-mini/Documents/code/agent-mesh/docs/components.md:96-108`). Hangar has no shipped inter-agent ask/task protocol and explicitly parks that as aspirational Phase 9 (`/Users/george-mac-mini/Documents/code/hangar/docs/phases/09-inter-agent.md:1-15`), so Merge would move the hard work into a Rust/PTY/SQLite product without making it easier. Layer would bind agent-mesh's structured headless protocol to a prompt-into-terminal API that returns `204 No Content` rather than typed task results (`/Users/george-mac-mini/Documents/code/hangar/backend/src/api/prompt.rs:17-38`).

## 2. ANSWERS

1. **The coordination layer is the hard part; the session substrate is a solved-but-secondary risk.** Hangar proves that PTY sessions, driver parsing, persisted events, and backend-restart reattachment are buildable (`/Users/george-mac-mini/Documents/code/hangar/docs/ARCHITECTURE.md:126-137`, `/Users/george-mac-mini/Documents/code/hangar/backend/tests/supervisor_restart.rs:1-10`, `/Users/george-mac-mini/Documents/code/hangar/backend/tests/supervisor_restart.rs:245-270`). But agent-mesh's open roadmap is dominated by coordination primitives: envelope, embedded bus/KV, sidecar heartbeat, coordinator lease eviction, CAS claims, ticket FSM, async ask verbs, role routing, blackboard, and P3 DAG/expert policy (`/Users/george-mac-mini/Documents/code/agent-mesh/docs/components.md:9-68`, `/Users/george-mac-mini/Documents/code/agent-mesh/docs/components.md:89-104`, `/Users/george-mac-mini/Documents/code/agent-mesh/ARCHITECTURE.md:264-287`). Hangar's Phase 9 asks for `POST /sessions/:id/task`, queued/running/succeeded/failed, routing by kind/labels, and a job tree; that is a sketch, not an implementation (`/Users/george-mac-mini/Documents/code/hangar/docs/phases/09-inter-agent.md:7-20`). Engineering risk therefore concentrates in agent-mesh's durable state and routing contracts, not in child process spawn.

2. **PTY is justified for human-watched terminals, not as agent-mesh's primary headless substrate.** Hangar's driver contract is explicitly PTY-byte oriented: `on_bytes` parses terminal output, `prompt` writes text plus carriage return to the PTY, and Claude Code falls back to terminal pattern matching when hooks fail (`/Users/george-mac-mini/Documents/code/hangar/docs/SESSION-PROTOCOL.md:140-164`, `/Users/george-mac-mini/Documents/code/hangar/docs/SESSION-PROTOCOL.md:172-195`). Current source scrapes status lines and tool names from ANSI-stripped TUI bytes (`/Users/george-mac-mini/Documents/code/hangar/backend/src/drivers/mod.rs:1-19`, `/Users/george-mac-mini/Documents/code/hangar/backend/src/drivers/claude_code.rs:305-317`). Agent-mesh has already verified headless structured `claude -p --output-format json` fields and a resident stream-json expert with retained session context (`/Users/george-mac-mini/Documents/code/agent-mesh/docs/spikes/M0-feasibility.md:49-103`, `/Users/george-mac-mini/Documents/code/agent-mesh/docs/decisions/DECISIONS.md:9-17`). Run experts as resident stream-json processes; reserve PTY/tmux only for CLIs with no structured streaming mode.

3. **Do not import supervisor + `SCM_RIGHTS` now; import the failure-mode tests and lifecycle discipline.** Hangar's fd-passing is real: the supervisor protocol sends and receives fds via `SCM_RIGHTS`, and hangard reattaches by `attach_fd` after supervisor spawn/list recovery (`/Users/george-mac-mini/Documents/code/hangar/backend/src/supervisor_protocol.rs:120-147`, `/Users/george-mac-mini/Documents/code/hangar/backend/src/main.rs:183-254`). But the current proof is backend restart with supervisor still alive, not whole-container or host survival (`/Users/george-mac-mini/Documents/code/hangar/backend/tests/supervisor_restart.rs:238-270`), and ADR-0017 says container kill resets sessions (`/Users/george-mac-mini/Documents/code/hangar/docs/decisions/0017-containerize-deployment.md:52-57`). Cheaper respawn + `--resume` + blackboard rehydrate loses work only for in-RAM context since the last persisted summary, in-flight partial tool work, and stdin/stdout messages not yet converted into a durable note/ticket event. The settling experiment is: kill a resident stream-json expert mid-answer, restart via `--resume` plus `mesh context`, and measure lost turns, latency, and answer quality against a supervisor-held process.

4. **Option D costs more than it saves.** "agent-mesh as hangar Phase 9" would require building the missing coordinator, task protocol, routing, retry/cancellation, and job tree inside hangar; hangar's own Phase 9 and Phase 10 docs mark inter-agent protocols and multi-node scheduling as aspirational (`/Users/george-mac-mini/Documents/code/hangar/docs/phases/09-inter-agent.md:1-15`, `/Users/george-mac-mini/Documents/code/hangar/docs/phases/10-multi-node.md:1-15`). The current hangar API route map has sessions, prompt, key, worktree, search, broadcast, metrics, costs, logs, and per-session PTY WS routes, but no task/ticket/blackboard/claim endpoints (`/Users/george-mac-mini/Documents/code/hangar/backend/src/api/mod.rs:46-92`). The Go+embedded-NATS premise also does **not** cleanly survive scrutiny: agent-mesh's newest decision log explicitly reopens transport and language, superseding the earlier embedded-NATS commitment (`/Users/george-mac-mini/Documents/code/agent-mesh/docs/decisions/DECISIONS.md:57-67`, `/Users/george-mac-mini/Documents/code/agent-mesh/docs/decisions/DECISIONS.md:81-89`). That weakens the brief's Go argument, but it does not rescue Merge; it means agent-mesh should decide transport on its own terms rather than inherit hangar's Rust/SQLite/PTY shape.

5. **If Rebuild, steal on day one: driver trait shape, worktree isolation, hook/OOB channel shape, and child lifecycle tests. Steal later: ring history, dashboard file/diff viewer, supervisor fd-passing.** Day-one steals: `AgentDriver`/`SpawnCfg`/`StateCtx` as the per-CLI adapter seam (`/Users/george-mac-mini/Documents/code/hangar/backend/src/drivers/mod.rs:224-298`); worktree-per-agent/tmux parallel isolation as an operational pattern (`/Users/george-mac-mini/Documents/code/hangar/.pipeline/parallel.sh:1-12`, `/Users/george-mac-mini/Documents/code/hangar/.pipeline/parallel.sh:89-107`, `/Users/george-mac-mini/Documents/code/hangar/docs/RUNBOOK.md:65-83`); hook-to-OOB-events as a typed side channel, but with real auth added (`/Users/george-mac-mini/Documents/code/hangar/backend/src/api/sessions.rs:196-223`, `/Users/george-mac-mini/Documents/code/hangar/backend/src/cc_hook_socket.rs:12-39`); reap-on-delete/child watchdog tests (`/Users/george-mac-mini/Documents/code/hangar/backend/src/pty.rs:22-49`, `/Users/george-mac-mini/Documents/code/hangar/backend/src/api/sessions.rs:322-346`). Later steals: bounded output history and ring/event split if agent-mesh needs replayable raw process logs (`/Users/george-mac-mini/Documents/code/hangar/docs/decisions/0011-ring-file-plus-events.md:11-25`, `/Users/george-mac-mini/Documents/code/hangar/docs/decisions/0012-100mb-history.md:17-26`), and supervisor fd-passing only after a headless expert crash test proves `--resume` is not good enough.

6. **If Layer, hangar's REST API is a poor programmatic substrate for agent-mesh.** Spawn now appears to honor `cwd`: the API validates absolute `cwd`, passes it into `SpawnRequest`, the Claude driver uses `project_dir.unwrap_or(req.cwd)`, and both portable-pty and supervisor paths set current directory (`/Users/george-mac-mini/Documents/code/hangar/backend/src/api/sessions.rs:69-87`, `/Users/george-mac-mini/Documents/code/hangar/backend/src/drivers/claude_code.rs:295-300`, `/Users/george-mac-mini/Documents/code/hangar/backend/src/pty.rs:311-320`, `/Users/george-mac-mini/Documents/code/hangar/backend/src/bin/hangar-supervisor.rs:309-340`). But programmatic fitness is still weak: `POST /prompt` writes text to a PTY and returns no typed completion (`/Users/george-mac-mini/Documents/code/hangar/backend/src/api/prompt.rs:17-38`); `POST /broadcast` sends prompts into all active sessions unless the caller supplies a filter (`/Users/george-mac-mini/Documents/code/hangar/backend/src/api/broadcast.rs:27-85`); backend auth is explicitly "Tailnet/Cloudflare Access is the boundary" and anyone reaching the API can type into sessions (`/Users/george-mac-mini/Documents/code/hangar/docs/decisions/0013-tailnet-only-auth.md:17-31`); event delivery is polling for structured events plus per-session PTY WS, not a durable cross-agent bus (`/Users/george-mac-mini/Documents/code/hangar/frontend/src/lib/stores/events.svelte.ts:214-233`, `/Users/george-mac-mini/Documents/code/hangar/backend/src/ws/pty.rs:49-97`). Backpressure is local and lossy: hook messages use a 64-slot channel and `try_send`, while PTY output uses a 256-slot broadcast channel that can lag clients (`/Users/george-mac-mini/Documents/code/hangar/backend/src/api/sessions.rs:196-203`, `/Users/george-mac-mini/Documents/code/hangar/backend/src/cc_hook_socket.rs:36-39`, `/Users/george-mac-mini/Documents/code/hangar/backend/src/pty.rs:326-327`, `/Users/george-mac-mini/Documents/code/hangar/backend/src/ws/pty.rs:65-67`).

7. **The missing question is: "What state must survive a crash, and what state may be sacrificed?"** Agent-mesh keeps saying "warm experts" and "durable blackboard," but those are different durability classes: the resident process holds RAM context, while the blackboard/ticket stream must hold decisions and work state (`/Users/george-mac-mini/Documents/code/agent-mesh/docs/decisions/DECISIONS.md:9-17`, `/Users/george-mac-mini/Documents/code/agent-mesh/docs/decisions/DECISIONS.md:33-41`). Before choosing any supervisor machinery, define the loss budget per entity: ticket accepted/unanswered, file claim, expert private context, worker patch, tool side effect, and dashboard transcript. Agent-mesh already encodes the right direction with one versioned envelope, typed results, TTL leases, and one KV authority (`/Users/george-mac-mini/Documents/code/agent-mesh/docs/decisions/DECISIONS.md:105-137`), but it needs an explicit durability matrix before P3.

## 4. STEAL / AVOID

**STEAL**

- **Driver seam per CLI** — `AgentDriver`, `SpawnRequest`, `SpawnCfg`, `StateCtx`, and `DriverRegistry` are the right adapter shape; replace PTY byte parsing with headless structured streams where available (`/Users/george-mac-mini/Documents/code/hangar/backend/src/drivers/mod.rs:224-298`, `/Users/george-mac-mini/Documents/code/hangar/backend/src/drivers/mod.rs:300-328`).

- **Reap-on-delete and lifecycle regression tests** — child processes need explicit reap/kill paths and tests because dropping a process handle is not enough (`/Users/george-mac-mini/Documents/code/hangar/backend/src/pty.rs:22-49`, `/Users/george-mac-mini/Documents/code/hangar/backend/src/api/sessions.rs:322-346`, `/Users/george-mac-mini/Documents/code/hangar/backend/tests/delete_reap_test.rs:1-11`).

**AVOID**
